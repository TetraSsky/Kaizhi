Création de la BDD :
CREATE DATABASE kaizhi_db
USE kaizhi_db;

Pour créer la table une fois la base de donnée créée : 
CREATE TABLE games
(
    gameid int NOT NULL, 
    titre text NOT NULL, 
    description text, 
    prix int, 
    image mediumblob, 
    liensteam blob, 
    PRIMARY KEY (gameid)
)

Vu que "gameid" est une clé primaire, on met en auto increment (Pour éviter réalistiquement des problèmes dans le futur avec des doublons) :
ALTER TABLE games MODIFY gameid INT AUTO_INCREMENT

Commande d'insertion de données (Générées par IA à partir de ce lien : https://api.steampowered.com/ISteamApps/GetAppList/v2 (API Steam JSON), au lieu de faire des call à celle-ci, même si cela était possible)

J'ai dû mettre en place un déclencheur au niveau la base pour "image" :
DELIMITER //
CREATE TRIGGER before_games_insert 
BEFORE INSERT ON games
FOR EACH ROW
BEGIN
    IF NEW.image IS NULL THEN
        SET NEW.image = '';
    END IF;
    IF NEW.liensteam IS NULL THEN
        SET NEW.liensteam = '';
    END IF;
END //
DELIMITER ;

Certaines données ont comme image=NULL ce qui ne correspond pas à une chaîne de caractère, d'où la nécessite de ce déclendeur
